### #1 📸 Маняша — чёрный квадрат в Safari
- **Причина:** маскот — `manyasha-idle-alpha.webm` (VP9 yuva420p). Safari **не поддерживает VP9/WebM** вообще → видео не декодируется, виден чёрный фон `<video>`. Текущий фолбэк (`src/hooks/useAlphaVideoFallback.ts`) рассчитан на Chrome-баг (canvas-проба + onError + таймаут); в Safari он может не срабатывать мгновенно (onError для неподдерживаемого кодека в Safari ненадёжен).
- **Места:** `src/components/Manyasha.tsx`, `src/components/ManyashaChat.tsx`, хук `useAlphaVideoFallback.ts`. Маскот на главной и «остальных экранах» (плавающий чат, страницы).
- **Фикс:** добавить в хук **детект поддержки кодека** `video.canPlayType('video/webm; codecs="vp9"')` — если пусто (Safari), сразу `fallback=true` (статичный `manyasha-idle-fallback.webp`). Не ждать onError/таймаут.
- **Acceptance:** в Safari (desktop+iOS) на всех экранах вместо чёрного квадрата — прозрачный кадр маскота.
- **Фото:** желателен скрин из Safari для подтверждения (чёрный именно квадрат, а не контейнер).

## БЛОК 2. Авторизация и 149-ФЗ (аудит выполнен)

**ГЛАВНОЕ: иностранных OAuth в коде НЕТ.** Ни Google/Apple/Facebook/GitHub/Discord/Microsoft, ни Telegram Login Widget. Нет NextAuth/Passport/Firebase/Supabase/Auth0. Авторизация **самописная** (`src/lib/auth.ts`: Node `scrypt` + opaque-сессии в БД, cookie httpOnly). **«Убирать кнопки гугла» нечего — их нет.**

Точки входа: email+пароль (`/api/auth/login`), вход через Telegram (deep-link + одноразовый код через `AuthCode`, **НЕ запрещённый виджет**), заглушка SMS (вкладка «скоро»).

### Соответствие ст.16 149-ФЗ
- ✅ Хеш паролей (scrypt+соль, timingSafeEqual), ✅ сессии httpOnly/secure/отзывыаемые, ✅ rate-limit входа, ✅ HSTS-заголовок.
- ❌ **Нет журнала попыток входа** (success/fail, IP, время) — добавить.
- ⚠️ **HTTPS-редирект** — `nginx/default.conf` слушает только :80; проверить, что edge-прокси редиректит 80→443 и шлёт `X-Forwarded-Proto`.
- ⚠️ **Осиротевший живой `/api/auth/register`** — UI не вызывает, но `POST` создаёт аккаунт без согласия на ПДн. Удалить файл или вернуть 405.
- ⚠️ Бот↔сайт: убрать legacy-fallback на `ADMIN_PASSWORD` в `src/lib/admin.ts:27-28`, оставить только `BOT_SHARED_SECRET`.

### Дополнительный метод входа — ВЫБРАН: Яндекс ID ✅
- **Решение пользователя: Яндекс ID** (OAuth 2.0, oauth.yandex.ru).
- **Объём работ (новая подсистема):**
  1. Регистрация приложения в Яндекс OAuth → `client_id` + `client_secret` (в `.env`, секрет только на сервере).
  2. Роут старта: редирект на `https://oauth.yandex.ru/authorize?response_type=code&client_id=...&redirect_uri=...`.
  3. Callback-роут `/api/auth/yandex/callback`: обмен `code`→`access_token` (POST oauth.yandex.ru/token), затем `login.yandex.ru/info` → email/имя/yandexId.
  4. Апсерт `User` по `yandexId`/email, создание сессии (переиспользовать `createSession` из `src/lib/auth.ts`).
  5. CSP: при необходимости расширить `connect-src`/`form-action` под `oauth.yandex.ru`, `login.yandex.ru` (`next.config.ts`).
  6. UI: кнопка «Войти через Яндекс ID» в `AuthForm.tsx` (заменить/дополнить вкладки).
  7. CSRF-защита `state`-параметром.
- **Замечание:** для 149-ФЗ это НЕ обязательно (email+Telegram уже легальны), но добавляем по решению.
